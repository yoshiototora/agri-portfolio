キウイ農家で雄花摘みを体験し、花粉を採るまでに多くの時間がかかることや、必要な日に人手を確保する難しさを知りました。

キウイ農家で農作業を体験した

2026年5月、農業のデイワークに参加し、キウイ農家で農作業を体験しました。普段キウイを食べているだけでは分からない、多くの手間がかかっていることを知りました。

雄花摘み： 花粉を採るために、雄花を一つひとつ摘む作業を手伝いました。約5時間作業しても、得られる花粉は数グラムほどでした。以前は受粉用の花粉を購入していたそうですが、価格が上がったため、現在は雄花を育てて自分たちで花粉を採っていると聞きました。

実際にやってみて気づいたこと

お世話になった農家は、高齢のご夫婦が営んでおり、人手不足のためデイワークを利用していました。直前のキャンセルが多く、「経験の有無よりも、予定通り人が来てくれること自体がありがたい」という言葉が特に印象に残っています。この農家では、必要な日に人手を確保することが大きな負担になっていると知りました。

農家の方から「アプリの文字が小さくて見づらい」と言われ、画面の見やすさも使いやすさに関わると感じました。その言葉が残り、このポートフォリオサイトにも、文字サイズやUDフォントを切り替えられる機能を付けました。

💡 この映像について

若年層にフィッシング詐欺の危険性を30秒で伝える啓発アニメーションです。

⚠️ 同世代にも被害が広がっていると知って

警察のサイバー防犯ボランティア活動（ネットパトロール等）に参加する中で、同世代にもフィッシング詐欺の被害が広がっていると知りました。ポスターやチラシだけでは興味を持ってもらいにくいため、普段SNSで流れてくる短い動画広告のような形で、危険性に気づく映像を作りたいと考えました。

🎬 若者の感覚に合わせた30秒の設計

若者が普段スマホで動画をスクロールする様子を意識し、親しみやすいキャラクターから始まり、途中で動画が巻き戻って「詐欺のポイント」を解説する30秒の構成をディレクションしました。映像内の偽サイト画面はHTML/CSSで自作して撮影し、本当にある偽画面の質感を再現しました。チーム内では全体の方向性を提案し、BGMや背景などの役割を分担しながら制作を進めました。

🌱 その後どうなったか

警察庁主催コンテストで全国1位（サイバー警察局長賞）を受賞しました。身近な友人からも「かわいらしい見た目なのに、最後のメッセージでドキッとした」と言われ、狙い通り危険性を伝えることができました。

📝 30秒でどう伝えるか

30秒という短い時間の中で、偽のメッセージを開き、リンクを押してしまうまでの流れをどう見せるか、3人のチームで何度も調整しました。普段のスマートフォン操作と重ねて見てもらえるよう、画面の見せ方や場面の順番を考えました。

大学院研究 / 修士課程（個人研究）

AIを用いた360°3D映像の境界破綻低減

少ないレンズ数と軽量な処理で、つなぎ目のズレが少ない360°3D映像を生成する手法を探る研究

役割・担当: 研究者（Unity実装、AIアルゴリズム構築、物理器具自作）
技術スタック: Python / PyTorch / OpenCV / Unity / 3Dプリンター
修士研究メインコード 立体映像評価用(vr-ssq-evaluation) Quest 3S 表示テスト用

💡 研究の内容

高価な専用機材を使わず、市販の360°カメラで360°3D映像を生成することを目指しています。卒業研究で見えてきたつなぎ目のズレや処理速度の問題を、AI技術を使って改善できないか試している段階です。

🔬 少ないレンズで撮影すると起きるつなぎ目の崩れ

少ないレンズ数で360°立体映像を撮影する場合、レンズのつなぎ目部分で映像が途切れたり、重なったりするズレ（境界破綻）が起きてしまいます。これを解決するため、AIを用いて境界部分を滑らかに合成する処理の開発に取り組んでいます。

🛠️ 試行錯誤していること

何から試せばよいか分からず、まず複数の深度推定モデル（Depth Anything V2など）を同じ映像で比べました。人物がつなぎ目を通る部分の合成方法を変える処理を作り、Quest 3Sで実際に映像を表示しながら、どの方法でズレが減るかを比較しています。また、撮影位置をそろえて同じ条件で比較できるよう、3Dプリンターでカメラの固定台も作りました。

📝 これまでに分かったことと課題

処理方法の違いを正確に比べるには、映像処理だけでなく、カメラの位置や撮影条件をそろえる必要がありました。現在はQuest 3Sで比較する環境を整え、視聴実験に向けた準備を進めています。

技術的詳細・試行内容

・深度推定による補完: Depth Anything V2で各フレームの深度マップを推定し、つなぎ目付近の視差補正に利用できるか検討中。

・Quest 3Sでの比較実験: 異なる補正方法を適用した映像をQuest 3Sで実際に視聴し、ズレや違和感を比較しています。

・物理固定台の自作: 3Dプリンターで独自のカメラ固定用リグを作製し、撮影時の物理的なブレをできるだけ抑えています。

💡 この研究について

市販の360°カメラ3台を組み合わせ、低コストで自然な360°3D映像を合成する研究です。

🔬 安価なカメラでのズレの発生

市販の安価な360°カメラを複数使って立体映像を撮影すると、映像のつなぎ目で人の体が途切れて見えたり、映像が不自然に揺れたりするズレ（境界破綻）が起きていました。

🛠️ ソフトとハードの両面からの工夫

映像の中の動く人物と背景を自動で区別し、つなぎ目を滑らかにつなぎ合わせるフェード処理プログラムを作成しました。また、カメラ同士の距離や角度がずれないよう、3Dプリンターで専用の固定台も設計・作成しました。

🌱 得られた検証結果

つなぎ目のズレを抑えた立体映像を再現できることを検証し、卒業論文として発表しました。

📝 手を動かして分かったこと

プログラムを調整するだけでは撮影条件によって結果が変わったため、カメラの位置をそろえる固定台も作りました。この経験から、映像処理だけを見るのではなく、撮影方法まで含めて確かめるようになりました。

個人開発 / iOSアプリ配信

iOSアプリ『HabitSpark』の開発と公開後の改善

一度できなくても挫折しない「失敗しても続けられる仕組み」と、レビューを踏まえた改善

役割・担当: 個人開発（企画・UI/UX・DB設計・実装・配信・アップデート保守すべて）
技術スタック: Swift / SwiftUI / SwiftData / Git
App Store (HabitSpark) GitHub (GoalTracker)

💡 アプリについて

「HabitSpark」は、できなかった日があっても、振り返りながら続けられる習慣化支援iOSアプリです。企画、デザイン、Swiftでの実装、App Storeでのリリースまで一貫して単独で行いました。

📱 習慣化で自分自身が挫折した経験

自分自身、日々の習慣を始めようとしても、一度できない日があるとそのままやめてしまうことがありました。

🛠️ 振り返りを次の行動につなげる仕組み

目標を月・週・日の3段階に分解できるようにし、失敗した日はKPT（Keep/Problem/Try）で振り返れる仕組みを作りました。さらに、App Storeのレビューで「できなかったタスクを翌日に引き継ぎたい」という意見をいただき、単純な繰り越しではなく、3日連続で未完了の場合は目標設定自体を見直せる仕組みを追加しました。

🌱 公開とユーザーからの反応

現在もApp Storeで公開しており、星5の高評価レビューをいただいています。届いた意見をもとに更新を続けています。

📝 レビューから機能の見直しへ

公開後、「未完了のタスクを翌日に持ち越したい」というレビューが届き、持ち越し機能を追加しました。自分だけで考えていたときには気づかなかった使い方を知り、公開後も機能を見直しています。
