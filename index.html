Background

農業を体験して分かったこと

キウイ農家での作業を通じて、作物づくりには、消費者から見えにくい多くの工程や工夫があることを知りました。また、自分の手で作物づくりに関わる面白さも実感しました。

きっかけ

「普段食べている作物は、どのように育てられているのだろう」という疑問から、農業デイワークに参加しました。その経験をきっかけに、家庭菜園も始めました。

現場で知ったこと

キウイ農家で、雄花摘みや受粉作業を経験しました。雄花を約5時間摘んでも、得られた花粉は数グラムほどでした。作物を育てるまでには、多くの工程と手間が必要であることを実感しました。

また、農作業には適した時期があり、必要な時期に人手を確保することの難しさも知りました。農家の方からは、予定していた人が1人来られないだけでも、作業が大きく遅れると伺いました。

その中で、農家の方から「農業デイワークのアプリは文字が小さく見づらい」と聞きました。情報を届ける際には、年齢や利用環境への配慮も必要だと学びました。

体験から考えたこと

自分で作業したことで、農業への理解を深めるには、情報を見るだけでなく、現場に関わる経験が重要だと感じました。

また、作物づくりにかかる手間だけでなく、自分の手で関わる面白さも実感しました。

約5時間で数グラム 花粉価格の上昇を受けた、自家採取のための雄花摘み 作物づくりを支える、消費者には見えにくい工程と工夫を知りました。
1人欠けるだけでも作業が大きく遅れる 適した時期に必要な人手を確保する難しさ 農作業では、必要な日に人が集まること自体が重要だと知りました。
Career Vision

農業分野で実現したいこと

農への関心を、行動と継続的な関わりへつなげたい。

私自身、農業デイワークが農を知る最初の一歩となり、家庭菜園へと関心が広がりました。一方、家庭菜園では、何をいつすればよいか分からず、続け方に迷いました。この経験から、農に関心を持つ人が自分に合った入口を見つけ、その後も関わりを続けられる仕組みを増やしたいと考えています。

そのために、農家や利用者の声から課題を整理し、社内外の関係者と連携しながら、企画の立案から実証、改善までを進める役割を担いたいです。Web開発や映像制作も、人と農との接点を広げる手段として活用します。

大学公式採用・公開準備中

長崎大学オープンキャンパス公式Webアプリ

高校生が展示を探しやすく、運営者も情報を更新しやすいWebアプリづくり

担当：課題発見、企画提案、画面設計、開発、教職員との調整
技術スタック：React / Next.js / TypeScript / Googleスプレッドシート / Git
大学公式プロジェクトのため、ソースコードは非公開です。2026年7月の公開に向けて準備を進めています。

1. 背景・課題

オープンキャンパスで研究室紹介を担当していた際、高校生から次に訪れる展示について尋ねられました。一緒に紙のマップを確認する中で、自分自身も目的の展示を探しにくいと感じました。

また、展示内容や場所が直前に変わった場合、紙のマップではすぐに対応できないことも課題でした。

2. 自分の役割

課題発見からアプリの企画提案、画面設計、有志メンバーとの役割分担、教職員との調整、中心となる開発業務まで担当しました。

3. 取り組み

スマートフォンで展示場所と内容を確認できるWebアプリを企画しました。高校生が学内を歩きながら、現在地に近い展示を探せるようにしています。

また、毎年プログラムを書き直さなくても済むよう、Googleスプレッドシートの内容をアプリに反映できる仕組みにしました。専門知識がない担当者でも、展示情報を更新しやすくするためです。

情報更新の流れ
1

展示内容をまとめる

2
アプリへ自動反映

更新の手間を減らす

3

展示場所を探しやすくする

4. 周囲との関わり

有志メンバー4名と役割を分担しながら開発を進めました。大学の広報を担当する教職員とも相談を重ね、安全基準やデザインについて調整しました。後輩へ技術を引き継げるよう、ドキュメント整備も進めました。

5. 結果

この提案は大学公式の企画として採用されました。現在は2026年7月のオープンキャンパスでの公開に向けて、展示情報の整理、画面調整、後輩へ引き継げる運用体制づくりを進めています。

6. 学び

開発するだけでなく、運用しやすい設計が大切だと学びました。また、教職員や有志メンバーなど立場の異なる人と対話し、合意をつくりながら進める経験にもなりました。

サイバー警察局長賞受賞

フィッシング詐欺対策啓発動画

若者に届く伝え方を考え、企画・構成・制作を主導した啓発映像（最優秀作品・サイバー警察局長賞）

体制：3名チーム
担当：企画、構成、ディレクション、映像制作
技術スタック：Blender / Adobe Premiere Pro / Git
YouTube（完成映像） 新しいタブで開く 警察庁 受賞詳細 新しいタブで開く
関連するメディア掲載を見る
長崎大学 NEWS掲載 新しいタブで開く 共同通信 記事 新しいタブで開く 読売新聞 記事 新しいタブで開く NBC長崎放送 記事 新しいタブで開く

1. 背景・課題

警察のサイバー防犯ボランティア活動に参加する中で、同世代の若者にもフィッシング詐欺の被害が広がっていると知りました。

ポスターやチラシだけでは興味を持ってもらいにくいため、SNSで流れる短い動画広告のように、自然に危険性へ気づける映像を作りたいと考えました。

2. 自分の役割

有志3名のチームで、ディレクター兼メインクリエイターとして、全体の企画立案、絵コンテ作成、進行管理、Blenderを用いた3Dアニメーション制作を担当しました。

3. 取り組み

若者が普段スマホで動画を見ている場面を意識しました。親しみやすいキャラクターから始まり、途中で動画が巻き戻って「詐欺のポイント」を解説する30秒の構成にしています。

著作権や安全面の制約を考え、映像内の偽サイト画面はHTML/CSSで自作しました。現実感のある画面を再現しながら、実在するサービスと混同されない表現にしました。

4. 周囲との関わり

チームメンバーとこまめに進捗を確認し合い、BGM、背景美術、3D制作などの役割を整理しながら進めました。ボランティア先の警察担当者からもフィードバックをいただき、防犯の意図が正しく伝わるか検証を重ねました。

5. 結果・反応

警察庁主催「令和7年度サイバー防犯ボランティア広報啓発コンテスト」フィッシング対策部門の最優秀作品に選ばれ、サイバー警察局長賞を受賞しました。

警察庁幹部からは「質の高い描画に加え、音声、効果音、全てにおいて手の込んだ丁寧な仕上がり」と評価されました。

6. 学び

防犯という硬いテーマでも、受け手の視点に立てば伝え方を工夫できると学びました。正解のない課題に対して、対象者を考え、制約を乗り越え、チームで成果まで導く経験にもなりました。

卒論発表済・大学院で発展中

低コストな360度3D映像生成の研究

学部で境界破綻を抑える手法を構築・検証し、大学院で視差条件の最適化へ発展させている研究

体制：個人研究
担当：手法設計、実装、実験環境構築、比較評価
技術スタック：Python / PyTorch / OpenCV / Unity / 3Dプリンター
研究コードを見る 新しいタブで開く
技術的な詳細リポジトリを見る
立体映像評価用 新しいタブで開く Quest 3S 表示テスト用 新しいタブで開く
Bachelor 深度推定・前景分離で境界破綻を改善
Master 視差パラメータの最適化
Impact 低コストな360度3D映像生成

1. 背景・課題

離れた場所の様子を、その場にいるような臨場感で伝えられる360度3D映像に可能性を感じ、研究テーマに選びました。一方、撮影には多くのレンズを備えた高価な専用機材が必要であり、利用できる場面が限られています。

そこで、市販の360度カメラを用いて、機材費を抑えながら立体的な360度映像を生成することを目指しています。

2. 自分の役割

研究テーマの設定から、合成アルゴリズムの設計・プログラミング、Unityを用いたHMD（Quest 3S）表示テスト環境の構築まで行っています。比較実験で使うカメラ固定台の設計と3Dプリントも担当しています。

3. 取り組み

学部研究では、複数の映像をつなぐ際、人物の輪郭が映像のつなぎ目で途切れたり、不自然に変形したりする課題に取り組みました。深度推定によって映像内の距離を捉え、前景分離で人物と背景を分けて処理する手法を構築し、比較実験を通じて効果を検証しました。

現在は大学院で、学部で構築した手法を基に、左右の映像のずれである視差の設定が立体感や見え方に与える影響を評価するため、Quest 3Sを用いた視聴比較実験を進めています。

4. 周囲との関わり

研究を進める上で、指導教員やゼミのメンバーに進捗を共有し、映像の違和感やつなぎ目の見え方について意見をもらいました。得られたアドバイスをもとに、人物の輪郭部分の補正アルゴリズムを改良しました。

5. 結果

学部研究では、深度推定と前景分離を用いて人物と背景を分けて処理することで、映像のつなぎ目における人物の輪郭の崩れを抑えられることを確認しました。現在は、Quest 3Sを用いた視聴比較実験を行い、視差の設定による立体感や見え方の違いを検証しています。

6. 学び

課題を捉え直し、異なる視点から解決策を考える大切さを学びました。また、ソフトウェア上の処理だけでなく、撮影条件を揃える固定台づくりまで含めて考えることで、結果を比較しやすい実験環境を構築できると学びました。

App Store公開・継続改善中

iOSアプリ「HabitSpark」

一度できなくても挫折しない「失敗しても続けられる仕組み」と、レビューを踏まえた改善

体制：個人運営
担当：企画、設計、開発、公開、利用者の意見を基にした改善
技術スタック：Swift / SwiftUI / SwiftData / Git
App Store (HabitSpark) 新しいタブで開く
開発コードを見る
GitHub (GoalTracker) 新しいタブで開く

1. 背景・課題

自分自身、新しい習慣を始めても、一度できない日があるとそのまま挫折してしまうことがありました。継続日数を重視する仕組みでは、一度できなかったときに意欲が下がりやすいと感じました。

2. 自分の役割

自身の悩みからアプリを企画し、UI/UX設計、データベースの設計、SwiftによるiOSアプリ開発、App Storeでの配信、リリース後のアップデート保守まで、個人として一貫して行っています。

3. 取り組み

目標を月・週・日の3段階に分けられるようにし、失敗した日はKPT（Keep/Problem/Try）で振り返れる仕組みを作りました。

レビューで寄せられた意見をもとに、未完了のタスクを翌日に持ち越せる機能も追加しました。

4. 周囲との関わり（公開後のユーザーの声）

App Storeのレビューや利用者からの意見を通じて、自分だけでは想定していなかった「未完了のタスクを翌日に持ち越したい」というニーズを把握しました。

要望をそのまま機能にするのではなく、習慣の挫折を防ぐというアプリの目的に照らして検討し、未完了が続いた場合には目標設定の見直しを促す導線も加えました。

5. 結果

App Storeでの公開後、利用者からレビューをいただきました。現在も、その意見を反映したアップデートを継続しています。

6. 学び

企画から公開までの一連の流れを自分の手で進める責任感を学びました。また、ユーザーの声を受け止め、柔軟にプロダクトを改善することの重要性も実感しました。
