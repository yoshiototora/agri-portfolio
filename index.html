Background & Vision

現場から課題を見つけ、仕組みで解決する

私は、現場に足を運び、利用者と運用者の両方が無理なく使い続けられる仕組みをつくることを大切にしています。システムの性能だけでなく、実際に使われ続けることを重視し、課題の発見から運用まで見据えた開発に取り組んでいます。

原体験：キウイ農家での農作業体験

2026年5月、農業のデイワークに参加し、キウイ農家での作業を経験しました。そこで、消費者からは見えにくい多くの地道な工程があることを知りました。

受粉作業： 受粉作業では、雄花から採取したわずか数グラムの花粉を、一つひとつの雌花へ手作業で噴射する工程を経験しました。消費者からは見えない地道な作業が品質を支えていることを知り、生産現場の負担の大きさを実感しました。この体験をきっかけに、後に自分でも家庭菜園を始めて野菜を育てるようになりました。

課題の認識：人手不足の深刻さとアプリの使いづらさ

お世話になった高齢のご夫婦農家は、人手不足のためデイワークを利用していました。しかし、直前のキャンセルが多く、「経験の有無よりも、予定通り人が来てくれること自体がありがたい」という言葉が特に印象に残っています。人手不足が単なる効率の問題ではなく、生産活動そのものに影響する深刻な課題であることを実感しました。

また、「アプリの文字が小さくて見づらい」という声も聞きました。現場を支える高齢の農家にとって、機能の多さよりも使いやすさが重要であり、利用者視点で設計する必要性を強く感じました。

ビジョン

農業現場で得た経験を通じて、課題は現場に足を運ばなければ見えてこないことを学びました。私は、利用者の声を起点に課題を発見し、技術によって解決策を仕組みとして実装することにやりがいを感じています。農業に関わる人々が本来の仕事に集中できる環境を、テクノロジーで支えたいと考えています。

アグリメディアで実現したいこと

アグリメディアでは、現場のリアルな課題を把握し、利用者と運用者の双方にとって使いやすいサービスづくりに取り組みたいと考えています。これまでのアプリ開発や研究活動で培った課題発見力と実装力を活かし、サービスの企画から改善まで一貫して関わりながら、現場で本当に使われる仕組みを形にしていきたいです。

💡 概要 / Overview

若い世代に身近なサイバー犯罪である「フィッシング詐欺」の危険性を、30秒で直感的に伝える啓発アニメーションです。普段からネット通販やSMSを使う同世代に向けて、「自分も被害に遭うかもしれない」と自分事として捉えてもらうことを目的に制作しました。

⚠️ 課題・応募の経緯 / Problem & Background

近年、ネット詐欺などのサイバー犯罪が身近な脅威になっていると感じていました。その中で、警察と連携して違法サイトの発見やネットパトロールを行う「サイバーテクニカルボランティア」の活動に参加し、実際に長崎県内でもフィッシング詐欺の被害が多発している現実を知りました。従来のポスターや講習会では若年層に関心を持たれにくいため、ボランティア活動で得た知識と自らのIT・映像技術を活かし、身近な危険性を同世代に発信したいと思いコンテストへの応募を決めました。

🛠️ 解決策と制作の工夫 / Solution & Creative Efforts

・親しみやすさと恐怖のギャップ： 最初の入り口をやわらかくするため、犬と猫の親しみやすいキャラクターで見やすくしつつ、その先で「日常のちょっとした油断から誰でも被害に遭いうる怖さ」をしっかり伝える構成にしました。

・直感的に伝わる30秒の構成： 短い時間で「被害に遭う流れ → 危険に気づくポイント → 防止策（URLを直接押さない等）」まで自然に繋がる構成にこだわりました。動画を途中で巻き戻す演出を入れることで、「どこで気づけたか」を視覚的に分かりやすく表現しました。

・HTML/CSSによる偽サイトの再現： 映像に登場する偽サイトの画面は、単なる画像ではなくHTMLとCSSで実際に自作・録画したもので、本当にありそうな見た目や動きをリアルに再現しました。

・完全オリジナルへのこだわり： 表情や目線、指の動きまで細かく調整し、1秒のアニメーションを作るのに3時間かけることもありました。効果音（海岸で録音した波の音）やキャラクターの声（声優の学校に通っていた兄や友人に協力してもらい収録）まで完全オリジナルで制作しました。

・ディレクターとしての分担： 制作期間約1か月半の中で、自身が全体のディレクション（企画・構成・演出）とメインのアニメーションを担当し、先輩2名に背景や効果音制作を依頼しました。少人数だからこそ密に連携を取り、それぞれの得意分野を活かしました。

🌱 成果 / Outcome

全国の大学や専門学校から集まった作品の中から第1位（サイバー警察局長賞）を受賞しました。家族や友人からも「かわいらしい見た目なのに内容はしっかり怖さが伝わる」と好評で、狙いであった「見やすさと危険性の両立」を社会に届けることができました。

💡 概要 / Overview

学部の卒業研究で見えてきた新たな課題を解決し、より実用的なシステムへと発展させるための研究です。

⚠️ 課題 / Problem

学部での研究終了時、立体感が強すぎて被験者が疲れやすいことや、カメラの手ブレ補正のズレによる映像の乱れが課題となりました。また、処理に1フレーム3秒かかり、リアルタイム配信に使えない点も問題でした。

🛠️ 解決策 / Solution

AIの処理速度を高速化し、リアルタイムでの映像合成と配信を可能にするプログラムの開発に取り組んでいます。また、目の疲れを和らげるために、見え方のズレを自動で補正する技術の導入や、手ブレによる映像のズレを自動で追いかける処理の構築を進めています。

🌱 成果 / Outcome

視聴者の目の負担を減らし、リアルタイムで安定して動く合成システムの確立を目指しています。現在は被験者による視聴実験を行い、アンケートや実験データを集めてVR酔いの低減効果を検証しています。

学部研究 / 卒業論文（個人研究）

簡易360°全周囲3D映像のシームレス合成手法

安価なカメラ3台を用いた低コストで自然な360度立体VR映像の再現

役割・担当: 卒業研究者（個人研究・プログラム設計・器具作成すべて）
技術スタック: Python / PyTorch / OpenCV / 3Dプリンター
卒業研究のメインコード

💡 概要 / Overview

市販の360度カメラ3台を組み合わせ、低コストで360度立体VR映像を合成する研究です。

⚠️ 課題 / Problem

360度カメラを複数使い、360度立体VR映像を合成する際、映像のつなぎ目で、人の体が切れて見える問題がありました。また、動画ではつなぎ目が揺れてしまうことも問題でした。これらがVR酔いを引き起こす大きな原因でした。従来の固定した修正方法では、人の動きや環境の変化に対応できませんでした。

🛠️ 解決策 / Solution

AI（深層学習）を用いて映像内の人物と背景を自動で切り分けることで、つなぎ目を滑らかにつなぐフェード処理を組み合わせた合成手法を開発しました。また、撮影時の物理的なカメラのズレを防ぐため、3Dプリンターで専用の固定台も自作しました。

🌱 研究成果と今後の展望 / Outcome & Future Work

映像のつなぎ目の乱れや揺れを抑えた、自然な360度立体VR映像の再現に成功しました。この成果を卒業論文として発表しました。また、開発したシステムを検証する中で、「立体視による目の疲れ」「手ブレ補正による余計なパーツの映り込み」「処理速度の遅さ（1フレーム約3秒）」という新たな3つの今後の課題を特定しました。

個人開発 / iOSアプリ配信

iOSアプリ「HabitSpark」のデータモデル再設計

ユーザーの「遡り入力」の要望を起点とした、データ構造の根本刷新

役割・担当: 個人開発（企画・UI/UX・DB設計・実装・配信・アップデート保守すべて）
技術スタック: Swift / SwiftUI / SwiftData / Git
App Store (HabitSpark) GitHub (GoalTracker)

💡 概要 / Overview

「HabitSpark」は、挫折を防ぐための習慣化支援iOSアプリです。企画、デザイン、Swiftでの実装、App Storeでのリリースまで一貫して単独で行いました。

⚠️ 課題 / Problem

既存の多くの習慣化アプリは「毎日完璧に継続すること」を前提としており、一度サボってしまうとモチベーションが途切れ、そのまま挫折しやすいという課題がありました。また、目標の粒度が大きすぎて、何から手をつければいいか迷う点も問題でした。

🛠️ 解決策 / Solution

目標を「月・週・日次」に段階的に分解できる設計にし、さらに失敗した日にはKPT（Keep/Problem/Try）法で振り返りを行います。Try（改善策）がそのまま翌日のやること（タスク）になる仕組みを作ることで、「挫折せず、自然に改善サイクルが回る」新しい習慣化アプリを開発しました。

⚙️ ユーザーフィードバックに基づくUX改善

毎日開きたくなる工夫としてヒートマップ表示などを設計する中、App Storeでユーザーから「できなかったTry（タスク）を翌日に引き継ぎたい」という意見をいただきました。そこで、単なるコピーではなく「Tryを翌日へ自動で引き継ぐ機能」を実装しました。また、3日間未完了が続いたTryは自動的に「Problem（課題）」へ差し戻し、目標設定の難易度自体を見直せる独自の仕組みを導入しました。これにより、「できなかったことへの罪悪感」を減らし、目標を適切に調整できる体験を目指しました。

🌱 成果 / Outcome

「Tryの自動引き継ぎ」や「Problemへの自動差し戻し」といったユーザーに寄り添ったUX設計が好評を得て、App Storeにて星5の高評価を獲得しました。
