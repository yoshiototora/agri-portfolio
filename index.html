Background & Vision

自分でやってみて分かったこと

キウイ農家で農作業を体験し、外から見ているだけでは分からない作業の手間や、農家の方が感じている不便を知りました。

キウイ農家で農作業を体験した

2026年5月、農業のデイワークに参加し、キウイ農家で農作業を体験しました。普段キウイを食べているだけでは分からない、多くの手間がかかっていることを知りました。

雄花摘み： 雄花を一つひとつ摘み、花粉を採る作業を手伝いました。約5時間かけても、得られる花粉はわずか数グラムでした。以前は受粉用の花粉を購入していましたが、価格が上がったため、この農家では雄花を育てて自分たちで採っていると聞きました。この体験をきっかけに、自分でも家庭菜園を始めましたが、何をいつすればよいのか分からないことが多くありました。

実際にやってみて気づいたこと

お世話になった農家は、高齢のご夫婦が営んでおり、人手不足のためデイワークを利用していました。直前のキャンセルが多く、「経験の有無よりも、予定通り人が来てくれること自体がありがたい」という言葉が特に印象に残っています。人を確保できるかどうかが、農園を続けるうえで大きな問題になっていると知りました。

農家の方から「アプリの文字が小さくて見づらい」と聞き、機能だけでなく、文字の読みやすさも大切だと感じました。この経験から、文字サイズ変更やUDフォントなど、読みやすさを意識した設計を取り入れています。

ビジョン

実際に足を運び、作業の苦労や使う人の声に直接触れる大切さを知りました。これからも、そこで気づいた不便さをもとに、誰もが使い続けられる仕組みを形にしていきたいです。

きっかけ

警察のサイバー防犯ボランティア活動に参加する中で、同世代の大学生がフィッシング詐欺でクレジットカード情報を入力してしまった被害を知りました。文章だけのチラシでは見てもらえないと考え、SNSの動画広告のようにスマホでつい目が止まる30秒のCG映像を作ろうと考えました。

やったこと

スマホをスクロールする手を止めるため、冒頭にかわいいキャラクターを出し、途中で動画が巻き戻って「詐欺のポイント」を解説する構成にしました。動画に登場する偽サイトの画面は、HTML/CSSを自分で書いてブラウザに表示したものを撮影し、本物の違和感（アドレスバーのドメイン表記の不自然さなど）を再現しました。3名のチームで、BGM担当や背景モデル担当に切り分けて分担し、ディレクションを行いました。

どうなったか

警察庁主催のコンテストで全国1位（サイバー警察局長賞）を受賞しました。身近な友人からも「かわいいキャラクターだったのに、最後に焦る様子があって怖さが伝わった」と言われ、伝えたいポイントが届いたことを確認できました。

今も考えていること

ポスターを貼るような従来の方法に頼らず、若者の実際のネット利用体験に合わせたストーリーを作る有効性がわかりました。今回の経験をもとに、SNSでさらに拡散されやすい短いコマ撮り風の表現など、次のアイデアを試しています。

大学院研究 / 修士課程（個人研究）

AIを用いた360°3D映像の境界破綻低減

少ないレンズ数と軽量な処理で、つなぎ目のズレが少ない360°3D映像を生成する手法を探る研究

役割・担当: 研究者（Unity実装、AIアルゴリズム構築、物理器具自作）
技術スタック: Python / PyTorch / OpenCV / Unity / 3Dプリンター
修士研究メインコード 立体映像評価用(vr-ssq-evaluation) Quest 3S 表示テスト用

きっかけ

自分が撮影した360度映像をVRゴーグルで見ると、人物がつなぎ目を横切る瞬間に体が半分に切れたり、不自然に膨らんだりするズレが見つかりました。高価なプロ用機材を使わず、市販のカメラでこれを解消したいと考えました。

最初に試したこと

何から試せばよいか分からず、まず複数の深度推定モデルを同じ映像で動かして処理の正確さを比べました。その中からDepth Anything V2を採用し、映像の奥行き情報（デプスマップ）から人物の位置を特定して、つなぎ目のブレンド具合を動的に変えるアルゴリズムを検証しています。

作って分かったこと

プログラムを書き換えても、撮影時の手ブレやカメラ自体の傾きがあると、合成時のズレが防ぎきれないことが分かりました。そこで、3Dプリンターで独自のカメラ固定用リグ（台座）を作り、カメラ同士の距離と角度を固定する物理的な対策も合わせました。

現在取り組んでいること

Quest 3Sを自分で被って合成した映像を見比べ、どの補正方法が最も自然に感じられるか、被験者へのアンケートデータも集めながらVR酔いの低減効果を分析しています。プログラムと物理リグの両面から、リアルタイムで処理できるレベルを目指して実験を続けています。

技術的詳細・試行内容

・深度推定による補完: Depth Anything V2で各フレームの深度マップを推定し、つなぎ目付近の視差補正に利用できるか検討中。

・Quest 3Sでの比較実験: 異なる補正方法を適用した映像をQuest 3Sで実際に視聴し、ズレや違和感を比較しています。

・物理固定台の自作: 3Dプリンターで独自のカメラ固定用リグを作製し、撮影時の物理的なブレをできるだけ抑えています。

学部研究 / 卒業論文（個人研究）

簡易360°3D映像のシームレス合成手法

安価なカメラ3台を用いた低コストで自然な360°3D映像の再現

役割・担当: 卒業研究者（個人研究・プログラム設計・器具作成すべて）
技術スタック: Python / PyTorch / OpenCV / 3Dプリンター
卒業研究のメインコード

きっかけ

安価な360度カメラを3台組み合わせて立体映像を撮ろうとしましたが、映像の境界線で人の体が途切れて見えたり、隣り合うカメラの明るさの違いでつなぎ目がはっきりと見えてしまったりしていました。

やったこと

動く人物の領域をプログラムで自動抽出し、背景との境界をグラデーションで馴染ませるブレンド処理を記述しました。さらに、3Dプリンターで3台のカメラのレンズ位置が等間隔で固定される専用台座を作りました。

どうなったか

つなぎ目のズレを抑え、市販の安価なカメラだけでも立体VRゴーグルで視聴できる映像を合成し、卒業論文としてまとめました。このとき見つかった「手ブレによる歪み」や「処理時間（1フレーム約3秒）」の課題が、現在の大学院での研究テーマに繋がっています。

作って分かったこと

高価な専用機材を購入しなくても、手元にある道具とプログラムを組み合わせれば工夫次第で映像の違和感を減らせることが分かり、自分の手で仕組みを作って改善を重ねる面白さを知りました。

個人開発 / iOSアプリ配信

iOSアプリ『HabitSpark』の開発と公開後の改善

一度できなくても挫折しない「失敗しても続けられる仕組み」と、レビューを踏まえた改善

役割・担当: 個人開発（企画・UI/UX・DB設計・実装・配信・アップデート保守すべて）

App Store (HabitSpark) GitHub (GoalTracker)

きっかけ

自分自身、家庭菜園の記録や日々のタスクで1日だけ書くのを忘れると、それだけでやる気が切れてしまう「完璧主義による挫折」を経験しました。失敗しても落ち込まずに再開できるアプリが欲しいと考えました。

やったこと

目標を月・週・日の3段階に細分化して行動しやすくし、できなかった日は「なぜできなかったか、次はどうするか」を振り返る振り返りカード（Keep/Problem/Try）を入力できる仕組みを設計しました。また、リリース後にApp Storeのレビューで「できなかった項目を翌日に引き継ぐ機能が欲しい」と書き込みがありました。

どうなったか

単にタスクを翌日へ丸投げするのではなく、3日連続で未完了のものは目標のハードルが高すぎると判定し、目標の再設定を促すアップデートを加えました。これにより「無理に続けさせない優しさがある」と継続ユーザーから好評を得ました。

今も考えていること

自分が「これが良いはずだ」と思って作った機能も、実際のユーザーが使い始めると異なる不便さ（自動繰り越しが欲しいなど）が生じることが分かりました。現在は、通知のタイミングを細かく調整できるアップデートに向けてコードを整理しています。
