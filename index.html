Background

自分で体験して分かったこと

キウイ農家での体験や、自分自身で家庭菜園を始める中で、消費者から見えにくい現場の負担と、作物づくりに関わる喜びの両方を知りました。

きっかけ

「普段食べている野菜や果物は、どのように育てられているのだろう」という疑問から、農業のデイワークに参加しました。

現場で知ったこと

キウイ農家で雄花摘みや受粉作業を経験しました。雄花を約5時間摘んでも得られる花粉は数グラムほどであり、作物ができるまでの手間を実感しました。また、必要な日に1人来ないだけでも作業が大きく遅れると伺い、人手確保の難しさを知りました。

また、農家の方から、農業デイワークのアプリは文字が小さく見づらいと聞き、情報を届ける際には、年齢や利用環境への配慮も必要だと学びました。

体験から考えたこと

農業には外から見えにくい負担がある一方、作物づくりに直接関わる喜びもあると知りました。農家を支えるだけでなく、農業の過程や魅力を知り、実際に関わる人を増やしたいと考えるようになりました。

将来実現したいこと

こうした現場や利用者の声を整理し、一人ひとりに合った農との接点を増やすサービス、企画、発信へつなげたいと考えています。

長崎大学オープンキャンパス公式Webアプリ

高校生向けマップ連動アプリの開発と、運営者の負担を減らす仕組みづくり

担当：課題発見、企画提案、画面設計、開発、教職員との調整
技術スタック：React / Next.js / TypeScript / Googleスプレッドシート / Git
大学公式プロジェクトのため学内限定公開

1. 背景・課題

オープンキャンパスで研究室紹介を担当していた際、高校生から次に訪れる展示について尋ねられ、一緒に紙のマップを確認しました。その際、自分自身も目的の展示を探しにくいと感じました。また、直前に展示内容や場所が変更された際、紙のマップではすぐに対応できないことが課題でした。

2. 自分の役割

3. 取り組み

スマートフォンで展示場所と内容をマップ上で確認できるWebアプリを企画しました。高校生が学内を歩きながら、現在地と近い展示をスムーズに探せるようにしました。また、毎年プログラムを書き直さなくても済むよう、Googleスプレッドシートの内容をアプリに反映できる仕組みにしました。これにより、専門知識がない担当者でも展示情報を手軽に更新・同期できます。

受賞実績／チーム制作

フィッシング詐欺対策啓発動画

警察の防犯ボランティアにおける若年層向け啓発映像制作（最優秀作品・サイバー警察局長賞）

担当：企画、構成、ディレクション、映像制作
技術スタック：Blender / Adobe Premiere Pro / Git
警察庁 受賞詳細 長崎大学 NEWS掲載 共同通信 記事 読売新聞 記事 NBC長崎放送 記事 YouTube（完成映像）

1. 背景・課題

警察のサイバー防犯ボランティア活動（ネットパトロール等）に参加する中で、同世代の若者にもフィッシング詐欺の被害が広がっていると知りました。ポスターやチラシだけでは興味を持ってもらいにくいため、普段SNSで流れてくる短い動画広告のような形で、危険性に気づく映像を作りたいと考えました。

2. 自分の役割

3名有志チームのディレクターとして、全体の企画立案、絵コンテ作成、進行管理、およびBlenderを用いた3Dアニメーション制作を担当しました。

3. 取り組み

若者が普段スマホで動画をスクロールする様子を意識し、親しみやすいキャラクターから始まり、途中で動画が巻き戻って「詐欺のポイント」を解説する30秒の構成をデザインしました。映像内の偽サイト画面はHTML/CSSで自作して撮影し、本当にある偽画面の質感を再現しました。

4. 周囲との関わり

チームメンバーとこまめに進捗を確認し合い、BGMのタイミングや背景美術の役割を分担しながら制作を進めました。ボランティア先の警察担当者からもフィードバックをいただき、防犯の意図が正しく伝わるか検証を重ねました。

5. 結果

警察庁主催「令和7年度サイバー防犯ボランティア広報啓発コンテスト」フィッシング対策部門の最優秀作品に選ばれ、サイバー警察局長賞を受賞しました。身近な友人からも「かわいらしい見た目なのに、最後のメッセージでドキッとした」と言われ、狙い通り危険性を伝えることができました。

6. 学び

伝えたい防犯という堅いメッセージを、受け手の視点（同世代の若者が好むテイスト）に立って伝わる構成に工夫すること、およびチームの得意分野を組み合わせて形にすることの意義を学びました。

大学院研究／比較検証中

低コストな360°3D映像生成と境界破綻の低減

市販のカメラを用いて機材費を抑え、つなぎ目のズレが少ない360°3D映像を生成する研究

担当：手法設計、実装、実験環境構築、比較評価
技術スタック：Python / PyTorch / OpenCV / Unity / 3Dプリンター
修士研究メインコード 立体映像評価用(vr-ssq-evaluation) Quest 3S 表示テスト用

1. 背景・課題

高価な専用機材と大きな処理負荷によって、360度3D映像の活用場面が限られています。そこで、市販 of 360度カメラを用いて機材費を抑え、少ない計算量で自然な映像を生成する手法を研究しています。

2. 自分の役割

研究テーマの設定から、合成アルゴリズムの設計・プログラミング、Unityを用いたHMD（Quest 3S）表示テスト環境の構築、比較実験用のカメラ固定台の設計・3Dプリントまで一貫して行っています。

3. 取り組み

少ないレンズ数で撮影する際、レンズのつなぎ目部分で映像が途切れたり重なったりする「境界破綻」が起きてしまいます。これに対し、映像の中の人物と背景を自動で区別して個別に合成処理を施す手法を考案しました。また、撮影時の物理的なブレや配置ズレによるノイズをなくすため、3Dプリンターで自作した専用のカメラ固定台を用いて撮影条件を均一に揃えました。

4. 周囲との関わり

研究を進める上で、指導教員やゼミのメンバーに進捗を共有し、映像の違和感やつなぎ目の見え方について意見をもらいました。得られたアドバイスをもとに、人物の輪郭部分の補正アルゴリズムを改良しました。

5. 結果

同一の映像と実行環境で比較し、従来手法より処理時間を短縮できることを確認しました。人物と背景を分けて扱う方法により、被写体の輪郭を保ちながら境界の不自然さを抑えることができました。現在はQuest 3Sを用いた視聴比較実験を進めています。

6. 学び

課題に対して原因を仮説検証し、ソフト（プログラム）とハード（固定台の自作）の両面からアプローチすることの有効性を学びました。また、客観的に評価するために「同一条件」で比較することの重要性を学びました。

公開・継続改善／個人運営

iOSアプリ「HabitSpark」

一度できなくても挫折しない「失敗しても続けられる仕組み」と、レビューを踏まえた改善

担当：企画、設計、開発、公開、利用者の意見を基にした改善
技術スタック：Swift / SwiftUI / SwiftData / Git
App Store (HabitSpark) GitHub (GoalTracker)

1. 背景・課題

自分自身、日々の習慣を始めようとしても、一度できない日があるとそのまま挫折してやめてしまうことがありました。世の中の多くの習慣化アプリは「毎日続けること」を重視するため、失敗した時点で意欲が下がりやすいことが課題でした。

2. 自分の役割

自身の悩みからアプリを企画し、UI/UX設計、データベースの設計、SwiftによるiOSアプリ開発、App Storeでの配信、リリース後のアップデート保守まで、個人として一貫して行っています。

3. 取り組み

目標を月・週・日の3段階に分解できるようにし、失敗した日はKPT（Keep/Problem/Try）で振り返れる仕組みを作りました。さらに、App Storeのレビューで「できなかったタスクを翌日に引き継ぎたい」という意見をいただき、単純な繰り越しではなく、3日連続で未完了の場合は目標設定自体を見直せる仕組みを追加しました。

4. 周囲との関わり（公開後のユーザーの声）

App Storeでの公開後、実際にアプリを使った方々のレビューに目を通し、直接意見をもらいました。自分だけでは想定していなかった「未完了タスクの翌日持ち越し」というニーズに対し、単に要望通りに引き継ぐだけでなく、挫折を防ぐための独自の緩和ルール（3日連続未完了時に警告）を提案して実装しました。

5. 結果

App Storeでの公開以来、現在も星5の高評価レビューをいただいており、利用者の声を反映したバージョンアップを継続的に実施しています。

6. 学び

自分の手で企画から公開までの一連の流れを進める責任感を体験し、かつユーザーの声（フィードバック）を受け止めて柔軟にプロダクトを改善・アップデートするプロセスの重要性を学びました。
